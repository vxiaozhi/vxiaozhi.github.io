# 详解圈复杂度

## 什么是圈复杂度？

圈复杂度(Cyclomatic complexity)是一种代码复杂度的衡量标准，在1976年由Thomas J. McCabe, Sr. 提出，目标是为了指导程序员写出更具可测性和可维护性的代码。

它可以用来衡量一个模块判定结构的复杂程度，数量上表现为独立路径条数，也可以理解为覆盖所有可能的情况最少需要的测试用例数量。 

## 意义

- 提前发现代码缺陷
- 具有更好的可测性&可维护性

## 代码圈复杂度的计算方法

通常采用的计算方法为点边计算法（当然还有节点判定法），计算公式为：

```
V(G) = e – n + 2
```
